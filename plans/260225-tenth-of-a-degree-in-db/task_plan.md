## Decisions Made
| Decision | Rationale |
|----------|-----------|
| Use Float/REAL for weather_data and forecast_snapshots temp columns | Matches existing hourly precision model and preserves one-decimal source data |
| Keep climate normals as Int and convert to Float when mapped into WeatherEntity | Climate normals are coarse historical averages and existing storage/logic assumes integer normals |
| Keep some historical charts/stats display integer-formatted by rounding at presentation edges | Avoid broad UI regression while preserving precise persisted data |
